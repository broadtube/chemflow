# Implementation Plan

- [x] 1. 基盤整備（エラー体系・成分レジストリ・グローバルFlowsheet）
- [x] 1.1 エラークラス体系の構築
  - chemflow固有の例外基底クラスと各サブクラス（示性式エラー、basis エラー、求解エラー、制約エラー、Canteraエラー）を定義する
  - 全エラーにわかりやすいメッセージを持たせ、原因特定が容易な構造にする
  - _Requirements: 1.2, 2.8, 6.3, 8.6_

- [x] 1.2 示性式から分子量を自動計算する成分レジストリの構築
  - molmassライブラリを使用し、示性式文字列（"H2", "CO2", "CH3COOH" 等）から分子量を自動計算する仕組みを作る
  - 同じ示性式に対する重複計算を避けるキャッシュ機構を設ける
  - 不正な示性式が渡された場合に適切なエラーを返す
  - 既存のComponentクラスは内部利用として維持し、ユーザーが直接触る必要をなくす
  - テスト: H2O=18.015, CO2=44.01, CH3COOH=60.052 等の分子量が正しいか検証
  - _Requirements: 1.1, 1.2_

- [x] 1.3 グローバルFlowsheetの仕組みの構築
  - モジュールレベルでFlowsheetインスタンスを管理し、全操作が暗黙的に登録される仕組みを作る
  - solve()、reset()、print_streams() をモジュールレベル関数として提供する
  - reset() で全ストリーム・全装置・全制約をクリアして新しい計算を開始できるようにする
  - _Requirements: 8.1, 8.4, 8.5, 8.6_

- [x] 2. Streamコンストラクタのリデザイン
- [x] 2.1 dict入力とbasis変換による直感的なストリーム定義
  - dict形式（`{"N2": 20, "H2": 60}`）でストリームを生成できるようにする
  - 示性式文字列をキーとして受け取り、成分レジストリ経由で分子量を自動解決する
  - basis="mol"（デフォルト）、"mass"、"normal_volume" の絶対量系3種に対応する
  - 重量流量やノルマル体積流量からモル流量への変換ロジックを実装する
  - 名前（name）はオプションとし、省略可能にする
  - グローバルFlowsheetへの自動登録を行い、値が確定しているストリームは自動的に固定扱いとする
  - テスト: 各basisで同一の物理的ストリームが等しいモル流量に変換されるか検証
  - _Requirements: 2.1, 2.2, 2.3, 2.9, 8.2_

- [x] 2.2 比率系basisの対応（モル比率・重量比率・体積比率）
  - basis="mole_frac"、"mass_frac"、"volume_frac" の比率系3種に対応する
  - totalパラメータが指定された場合は比率×合計からモル流量を算出して固定ストリームとする
  - totalが省略された場合は組成情報のみ保持し、合計流量を未知数として扱う（組成制約を自動登録）
  - テスト: 比率+totalで正しいモル流量が算出されるか、totalなし時に組成制約が登録されるか検証
  - _Requirements: 2.4, 2.5, 2.6, 2.7, 8.3_

- [x] 2.3 成分ごとの単位混在指定とCSV読み込み
  - タプル形式（`{"N2": (20, "mol"), "H2": (120, "mass")}`）で成分ごとに異なる単位を指定できるようにする
  - 絶対量系（mol, mass, normal_volume）のみ混在可能とし、比率系との混在はエラーにする
  - CSVファイルから成分名と流量を読み込んでストリームを生成する機能を実装する
  - _Requirements: 2.8, 2.10_

- [x] 2.4 未知ストリームと組成参照ストリームの定義
  - 成分リストのみを指定して未知ストリーム（全値が求解対象）を生成できるようにする
  - 他のストリームと同じ組成を持つが合計流量が未知のストリームを、compositionパラメータで生成できるようにする
  - composition指定時は組成一致の制約条件を自動的にFlowsheetに登録する
  - テスト: 未知ストリームが変数として扱われるか、composition指定で正しい制約が登録されるか検証
  - _Requirements: 2.7, 2.11_

- [x] 2.5 Streamの既存プロパティを新構造に適合させる
  - モル流量・重量流量・ノルマル体積流量のプロパティを新しいStream構造で動作するよう調整する
  - モル比率・重量比率・体積比率のプロパティを維持する
  - 個別ストリームの全情報を参照できることを確認する
  - _Requirements: 9.2_

- [x] 3. Expression レイヤーの実装（演算子による装置接続）
- [x] 3.1 遅延評価の基盤と混合演算子（+）の実装
  - ストリーム同士の加算（`A + B`）で遅延評価の混合Expression を返す仕組みを作る
  - 3つ以上のストリームの連続加算（`A + B + C`）にも対応する
  - Expressionが変数に代入されてStreamとして使われた時に、自動的にmaterialize（Stream生成 + Mixer登録）する
  - 異なる成分セットのストリーム同士が混合された場合、全成分の和集合を取り、存在しない成分は0として扱う
  - テスト: 2ストリーム混合、3ストリーム混合、異種成分混合の物質収支検証
  - _Requirements: 3.1, 3.2, 3.4_

- [x] 3.2 分割演算子（*）の実装
  - ストリームとスカラーの乗算（`A * 0.4`）で遅延評価の分割Expressionを返す仕組みを作る
  - 右辺からの乗算（`0.4 * A`）にも対応する
  - materialize時にSplit残差式を自動登録する
  - テスト: 分割比率で正しく物質収支が成立するか検証
  - _Requirements: 3.3_

- [x] 3.3 ExpressionのStreamプロキシ機能
  - Expressionオブジェクトに対してStreamのプロパティやメソッド（react、gibbs_react等）が呼ばれた場合、自動的にmaterializeしてからStreamに委譲する
  - `D = (A + B + C).gibbs_react(...)` のようなチェーン呼び出しが動作することを確認する
  - _Requirements: 3.1, 3.2_

- [x] 4. eq() と constrain() の実装
- [x] 4.1 等式制約関数 eq() の実装
  - `eq(target, expression)` で、Expressionを既存のStreamに紐付けて残差式を登録する仕組みを作る
  - Expressionが新しいStreamを作らず、target Streamに対する制約として登録されることを保証する
  - `eq(C, A + B)` と `eq(D, E_out + B)` の両方が正しく動作することを検証する
  - _Requirements: 4.1, 4.2_

- [x] 4.2 任意制約関数 constrain() の実装
  - lambda式を受け取り、残差関数としてFlowsheetに登録する仕組みを作る
  - 合計流量の固定（`lambda: C.total_molar_flow - 30`）が動作することを検証する
  - ストリーム間の流量等式（`lambda: A.total_mass_flow - E.total_mass_flow`）が動作することを検証する
  - 組成一致（`lambda: D.mole_fractions - E.mole_fractions`）のように配列を返す制約も扱えることを検証する
  - _Requirements: 7.1, 7.2, 7.3_

- [x] 5. 転化率指定リアクター（react メソッド）の実装
- [x] 5.1 化学量論リアクターのStream.react()メソッド実装
  - dict形式の化学量論係数（`{"CO": -2, "H2": -2, "CH3COOH": 1}`）と基準成分・転化率を受け取る
  - 入口ストリームに存在しない生成物成分を出口ストリームに自動追加する
  - 基準成分を示性式文字列（key="CO"）で指定できるようにする
  - 物質収支の残差式を自動的にFlowsheetに登録し、出口ストリームを返す
  - テスト: 単純反応（A → B 転化率80%）および複数成分反応の物質収支を検証
  - _Requirements: 5.1, 5.2, 5.3_

- [x] 6. Gibbsリアクター（Cantera連携）の実装
- [x] 6.1 Cantera平衡計算の統合とgibbs_react()メソッドの実装
  - 温度（°C）と圧力（Pa or 文字列）と平衡種リストを受け取り、ギブズ自由エネルギー最小化による平衡計算を行う
  - 摂氏→ケルビンの温度変換を行う
  - 圧力文字列のパース（"2MPaG" → ゲージ圧 → 絶対圧変換、"10atm" → Pa変換）を実装する
  - gri30.yamlから指定species をフィルタしてCantera Solutionを構築する
  - 平衡計算結果を残差式として全体の連立方程式に統合する（残差 = 出口 - 平衡値）
  - 入口に存在しない出口成分を自動追加する
  - テスト: CH4 + H2O 系の水蒸気改質平衡が化学的に妥当な結果を返すか検証
  - _Requirements: 6.1, 6.2, 6.3, 6.4, 6.5_
